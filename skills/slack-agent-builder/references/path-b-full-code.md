# Path B — Full-Code Build Steps (Claude Code + Vercel + Neon)

Step-by-step guide for building a production-ready Slack-native AI agent using Bolt for JavaScript, deployed to Vercel, with Neon for memory. Designed for execution in Claude Code.

---

## Before You Start

Confirm the user has:
- Claude Code installed and running
- A Vercel account (connected to their GitHub)
- A Neon account (neon.tech) with a project ready
- A Slack workspace where they can install apps
- An Anthropic API key

---

## Step 1: Install the Vercel Slack Agent Skill

In the Claude Code terminal, run:

```bash
npx skills add vercel-labs/slack-agent-skill
```

When prompted, install it to Claude Code. This gives Claude Code deep knowledge of Bolt, Chat SDK, Vercel deployment patterns, and Slack app configuration — it becomes the expert that guides the scaffold.

**Verbatim Claude Code prompt to paste:**
```
Install the Vercel Slack Agent Skill so we can build a Slack-native AI agent. Run: npx skills add vercel-labs/slack-agent-skill and install it to Claude Code.
```

---

## Step 2: Scaffold the Agent

In Claude Code, run the wizard:

```
/slack-agent new
```

When the wizard asks what you're building, describe the agent using the Agent Design Document from Phase 3. Be specific — the more detail, the better the implementation plan.

Example description format:
```
A [agent name] that [core job]. It lives in the [#channel] Slack channel and [how it's triggered]. It needs to [key capabilities]. Users are [audience description].
```

Select **Bolt for JavaScript** as the framework when prompted.

**Verbatim Claude Code prompt to paste:**
```
Run /slack-agent new. We're building [paste the Agent Design Document here]. Use Bolt for JavaScript as the framework.
```

---

## Step 3: Review and Approve the Implementation Plan

The wizard will generate an implementation plan showing:
- Core features
- Slash commands (if any)
- Event handlers
- File structure
- State management approach

Review it against the Agent Design Document. Request changes if needed. Confirm when ready.

**What to check:**
- Does the state management match Neon? (The default may be Redis/Upstash — change it to Neon)
- Are all the event handlers correct? (app_mention, message.im, etc.)
- Does the file structure make sense?

**Verbatim Claude Code prompt if state needs changing:**
```
Change the state management from Redis/Upstash to Neon (serverless Postgres). I use Neon as my primary database. Use the @neondatabase/serverless package.
```

---

## Step 4: Create the Slack App

Go to https://api.slack.com/apps

1. Click **Create New App** → **From a manifest**
2. Select your workspace
3. Paste the manifest generated by the wizard (it will be in the project files or shown in the wizard output)
4. Click **Create**
5. Go to **Install App** → **Install to Workspace** → Allow
6. Copy the **Bot User OAuth Token** (starts with `xoxb-`)
7. Go to **Basic Information** → copy the **Signing Secret**

---

## Step 5: Configure Environment Variables

Create `.env.local` in the project root:

```bash
SLACK_BOT_TOKEN=xoxb-[your-token]
SLACK_SIGNING_SECRET=[your-signing-secret]
ANTHROPIC_API_KEY=sk-ant-[your-key]
NEON_DATABASE_URL=postgresql://[your-neon-connection-string]
```

Get your Neon connection string from:
neon.tech → Your Project → Connection Details → Connection string (use the pooled connection)

**Verbatim Claude Code prompt:**
```
Set up the .env.local file with these environment variables: SLACK_BOT_TOKEN, SLACK_SIGNING_SECRET, ANTHROPIC_API_KEY, and NEON_DATABASE_URL. I'll provide the values.
```

## Step 6: Set Up Neon Memory Schema

Run the migration to create the conversation memory tables.

**Verbatim Claude Code prompt:**
```
Create a Neon database migration that sets up the conversation memory schema for our Slack agent. Include:
- slack_conversations table (id, channel_id, thread_ts, role, content, created_at)
- An index on (channel_id, thread_ts) for fast history retrieval
- A slack_agent_context table for storing agent-specific key/value data per channel

Run the migration against our Neon database using NEON_DATABASE_URL.
```

## Step 7: Write the System Prompt and Persona

This defines who the agent is. It should live in `lib/agent.ts` or `lib/system-prompt.ts`.

**Verbatim Claude Code prompt:**
```
Create the system prompt for our [Agent Name] in lib/agent.ts. The agent's persona is: [paste persona from Agent Design Document]. It lives in the [#channel] channel. Its core job is [core job]. Include instructions for how to format responses for Slack (concise, bullet points, no markdown headers — Slack renders these oddly).
```

---

## Step 8: Configure Event Handlers

Wire up the Bolt event handlers to the agent logic.

**Verbatim Claude Code prompt:**
```
Set up the Bolt event handlers for our Slack agent:
1. app_mention — when someone @mentions the bot in a channel
2. message.im — when someone DMs the bot directly (if needed)
[Add any slash commands from the Agent Design Document]

Each handler should:
1. Extract the message text, channel_id, and thread_ts
2. Call our handleMessage function from lib/bot.ts
3. Post the response back to the thread using say()
4. Handle errors gracefully (log them, post a friendly error message to the user)
```

---

## Step 9: Test Locally with ngrok

Open two terminal windows:

Terminal 1 — Start the dev server:
```bash
npm run dev
```

Terminal 2 — Start ngrok:
```bash
ngrok http 3000
```

Copy the `https://xxx.ngrok.app` URL from ngrok.

In the Slack App settings:
1. Go to **Event Subscriptions**
2. Enable Events
3. Set Request URL to: `https://xxx.ngrok.app/api/slack/events`
4. Wait for the green "Verified" checkmark
5. Save changes

Test in Slack:
- Go to the agent's channel
- Send a message or @mention the bot
- It should respond within a few seconds

**Common issues:**
- **No response:** Check the ngrok terminal for incoming requests. Check the dev server terminal for errors.
- **3-second timeout errors:** The Vercel Bolt adapter handles this, but make sure you're using it correctly.
- **Infinite loop:** Add a check at the top of every message handler: `if (message.bot_id) return;`

## Step 10: Deploy to Vercel

**Verbatim Claude Code prompt:**
```
Deploy this Slack agent to Vercel. Make sure:
1. The vercel.json has maxDuration: 30 for the Slack events function
2. All environment variables are configured in Vercel (same as .env.local)
3. The build completes without TypeScript errors
Deploy and give me the production URL.
```

After deployment, add the environment variables in the Vercel dashboard:
- Go to your Vercel project → Settings → Environment Variables
- Add all 4 variables from Step 5

---

## Step 11: Update Slack App Webhook URL

1. Go to https://api.slack.com/apps → your app
2. Go to **Event Subscriptions**
3. Change the Request URL from the ngrok URL to:
   `https://[your-vercel-domain].vercel.app/api/slack/events`
4. Wait for "Verified" → Save

If your app uses slash commands or interactivity:
- Go to **Slash Commands** → update each command's Request URL
- Go to **Interactivity & Shortcuts** → update the Request URL

---

## Step 12: Test the Full Conversation Loop in Production

Test all of the following:

1. Send a message in the Slack channel — does the agent respond?
2. Send a follow-up message — does it remember the context from the first message?
3. Start a new thread — is it a fresh conversation (no bleed from the previous thread)?
4. Check Neon — are messages being written to the `slack_conversations` table?

**Verbatim Claude Code prompt for verification:**
```
Query our Neon database to verify conversation memory is working. Show me the last 10 rows from the slack_conversations table, ordered by created_at DESC.
```

---

## Step 13: Final Bug Check

The agent is NOT done until:

- [ ] Responds to messages in the Slack channel without errors
- [ ] Remembers conversation context within a thread
- [ ] Starts fresh in a new thread
- [ ] No errors in Vercel function logs (check Vercel dashboard → Functions tab)
- [ ] No infinite loops (bot not responding to itself)
- [ ] Tested at least 5 messages in a full conversation loop

**Verbatim Claude Code prompt for log check:**
```
Check the Vercel function logs for our Slack agent deployment. Look for any errors, 500 responses, or timeout warnings. Show me the last 20 log entries.
```

## Troubleshooting Quick Reference

| Problem | Fix |
|---|---|
| Bot not responding in production | Check Vercel function logs, verify webhook URL is updated |
| "dispatch_failed" in Slack | Your function is timing out — check maxDuration in vercel.json |
| Bot responding to itself | Add `if (event.bot_id) return;` at top of handler |
| Memory not persisting | Check NEON_DATABASE_URL in Vercel env vars, check migration ran |
| Duplicate responses | Add deduplication check on event.ts field |
| 403 Unauthorized | Bot is not in the channel — run `/invite @botname` |
